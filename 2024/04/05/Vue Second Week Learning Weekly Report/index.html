一.Vue-cli

1.1 .什么是vue-cli

vue-cli官方提供的一个脚手架（预先定义好的目录结构及基础代码），用于快速生成一个vue的项目模板

1.1.1 .主要功能

  • 本地调试
  • 热部署
  • 单元测试
  • 集成打包上线
1.1.2 .环境准备
  • 安装Node.js（以管理员身份打开命令提示符cmd）
    • 检查Node是否安装成功 node -v
    • 检查npm npm -v

  • 安装vue-cli npm install vue-cli -g
    • 检查是否安装成功 vue list

二.Webpack

2.1.webpack简介

2.1.1.简介

​ 本质上，webpack是一个现代JavaScript应用程序静态模块打包器(module bundler)。当webpack处理应用程序时，它会递归地构建一个依赖关系图，其中包含应用程序需要的每一个模块，然后将所有的模块打包成一个或多个bundle。

2.1.2.现状

​ 当今越来越多的网站已经从王爷模式进化到WebApp模式。它们运行在现代浏览器里，使用HTML5、CSS3、ES6等技术来开发丰富功能，网页已经不仅仅是完成浏览器的基本需求。

​ WebApp通常是一个SPA（单页面应用），每一个视图通过异步的方式加载，这导致页面初始化和使用过程中会加载越来越多的JS代码，给前端的开发流程和资源组织带来了巨大挑战。

2.3.1. 配置

创建webpack.config.js配置文件

  • entry:入口文件，指定WebPack用哪个文件作为项目的入口

  • plugins: 插件，如：热更新，代码重用等
  • resolve：设置路径指向
  • ``watch：监听，用于设置文件改动后直接打包`

三.vue-router

3.1.介绍

vue-router是Vue.js官方的路由插件，它和vue.js是深度集成的，适合用于构建单页面应用。

vue-router是基于路由和组件的\n\n路由用户设定访问路径的，将路径和组件映射起来。\n在vue-router的单页面应用中，页面的路径的改变就是组件的切换

3.2 .安装

vue-router是一个插件包，在文件目录下直接npm i vue-router 是下载最新版，最新版不兼容vue2。所以我们要下载兼容版本 npm i vue-router@3.5.2，此时在package.json文件中可以看到 "vue-router": "^3.5.2"表示安装成功。

3.3.引入

如果在一个模块化工程中使用它，必须通过vue.use()明确安装路由功能

3.1.3 . 安装依赖

需要安装vue-router、element-ui、sass-loader、node-sass四个插件

3.1.3.1.安装vue-router

此时遇到了一个问题npm ERR!，这是因为我的项目中有两个不兼容的Vue版本引起的，此项目使用的是Vue 2.5.2，但是vue-router需要的是Vue 3.2.0或更高版本。我的package.json是 "vue-router": "^4.3.0"

解决方案

  1. 第一种方法是升级项目中的 Vue 版本，使其与 vue-router 的要求匹配。可以在项目的 package.json 文件中将 Vue 的版本号改为 3.2.0 或更高版本，然后重新运行 npm install 命令来安装依赖。如果选择升级 Vue 版本，可能还需要检查一下项目中是否有其他依赖项也需要相应的升级。

  2. 第二种方法是降级 vue-router 的版本，使其与项目中的 Vue 版本匹配。可以在项目的 package.json 文件中将 vue-router 的版本号改为适用于 Vue 2.x 的版本，然后重新运行 npm install 命令。

  3. 第三种方法是直接安装低版本的 vue-router :

    npm install vue-router@^3.5.2

这边我选择第三中解决方案
