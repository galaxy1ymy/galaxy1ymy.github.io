七.组件基础

组件允许我们将 UI 划分为独立的、可重用的部分，并且可以对每个部分进行单独的思考。在实际应用中，组件常常被组织成层层嵌套的树状结构，说白了就是一组可以重复使用的模板

  • 使用Vue.component()注册组件
  • 使用props传递数据
  • v-bind 用于动态地将数据绑定到 HTML 元素的属性上。它的作用是在 HTML 元素上绑定一个或多个属性，并将其与 Vue 实例中的数据进行关联。通常情况下，v-bind 用于绑定 HTML 元素的 class、style、src、href 等属性。

八.计算属性

简单说计算属性就是“将计算结果缓存起来变成一个静态的属性”

  • 在Vue模板中，调用方法需要在方法名后面加上括号 ()，而调用计算属性则不需要加括号。

  • 计算属性的语法

    computed: {
    Name: function() { // 计算属性的名称
    // 返回计算属性的值
    return /* 计算属性的值 */;
    }
    }

  • 第一个 <p> 元素用于显示通过方法获取的当前时间。

    第二个 <p> 元素用于显示通过计算属性获取的当前时间。

九.插槽内容

<slot> 元素是一个插槽出口 (slot outlet)，标示了父元素提供的插槽内容 (slot content) 将在哪里被渲染。

定义一个名为 todo 的自定义组件，该组件具有两个插槽：todo-title 和 todo-items。

以下是代码的分析：

  1. HTML 结构：
    • 有一个 Vue 应用程序的根元素 <div id="app">。
    • 在 <div id="app"> 中，有一个自定义组件 <todo>。
  2. Vue 组件：
    • 定义了三个 Vue 组件：
      • todo 组件：作为包含插槽的容器，其中包含两个插槽：todo-title 和 todo-items。
      • todo-title 组件：用于显示待办事项的标题。
      • todo-items 组件：用于显示待办事项的内容。

十.自定义事件

在组件的模板表达式中，可以直接使用 $emit 方法触发自定义事件 (例如：在 v-on 的处理函数中)：

<button @click="$emit('someEvent')">click me</button>

代码解析：

  • this.items 是 Vue 实例中的一个数据属性，它是一个数组，存储了待办事项列表中的所有事项。

  • splice(index, 1) //删除方法

    方法是在 this.items 数组上调用的。它接受两个参数：

    • index：指定了要删除的元素的索引位置。
    • 1：指定了要删除的元素个数，表示只删除一个元素。可以省略不写。
